System Overview

Precision dual-channel linear driver for the Mitsubishi ML20550KK14Z600L 1550 nm DFB butterfly laser module. Two independent control channels run on a Teensy 4.1 MCU via separate PID and ACC firmware loops: a constant-current ACC laser driver and a bipolar linear TEC temperature controller.

Laser Channel — ACC Driver

Low-side N-channel MOSFET current source with LT1028 error amplifier in a closed feedback loop. The DAC set-point drives the op-amp non-inverting input through a precision resistor scaler; the sense voltage across R5 feeds back to the inverting input. The loop regulates current regardless of laser diode forward voltage variation.

Full-Scale Calculation

Soft-Start

Implemented in Teensy firmware — ramp DAC code 0 → set-point over 5 ms at startup and after any fault condition. Never write a non-zero code without first confirming PGOOD high from the TPS7A39.

03

TEC Channel — Bipolar Linear H-Bridge

Four-quadrant linear H-bridge providing −1.5 A (heating) to +2.5 A (cooling) through the butterfly package internal Peltier element. Linear topology chosen specifically to avoid switching noise coupling into laser temperature through shared thermal mass.

Virtual ground: The AD5064 DAC outputs 0–5V unipolar. A precision resistor divider (R19 = 12.7 kΩ, R20 = 9.31 kΩ) from the 5V_DAC rail creates a TEC_VZERO reference at 1.875V. DAC code 24576 = 0A TEC current. Codes below = heating, codes above = cooling.

±15V Tracking — TPS7A39

The TPS7A39 is a monolithic dual LDO (positive + negative) sharing a single internal reference voltage. Both error amplifiers compare their feedback against the same Vref — ratiometric tracking is a mathematical consequence, not a separate circuit. The LM7815/LM7915 are not used — the TPS7A39 replaces them.

How Tracking Works

// Both outputs are slaves to the same Vref VOUTP = Vref × (1 + R1P/R2P) = Vref × 12.9 // positive VOUTN = −Vref × (1 + R1N/R2N) = −Vref × 12.9 // negative // At startup: NR/SS cap charges via 2µA → Vref ramps 0→1.2V // At Vref = 0.6V (halfway): // VOUTP = +0.6 × 12.9 = +7.74V // VOUTN = −0.6 × 12.9 = −7.74V ← simultaneously // The rails are always equal and opposite at every instant

Feedback Resistor Calculation

Target VOUTP = +16.5V (gives LT3045 1.5V headroom above +15V) VOUTP = 1.2 × (1 + R1P/R2P) 16.5 = 1.2 × (1 + R1P/10k) R1P = 10k × (16.5/1.2 − 1) = 10k × 12.75 = 127.5kΩ Use 127kΩ 0.1% → VOUTP = 1.2 × 13.7 = 16.44V ✓ Same ratio for negative: R1N = 127kΩ → VOUTN = −16.44V ✓

// AD5064 — all four channels in use dac_write(0x1, laser1_code); // Ch A 0x13xxxx → Laser 1 ACC dac_write(0x2, tec1_code); // Ch B 0x23xxxx → TEC 1 dac_write(0x3, laser2_code); // Ch C 0x33xxxx → Laser 2 ACC ← new dac_write(0x4, tec2_code); // Ch D 0x43xxxx → TEC 2 ← new

LTC6655-5 — Precision Reference

The LTC6655-5 serves exclusively as the AD5064 voltage reference — its output drives REFA (pin 13) and REFB (pin 14) only, through the RC Vref filter. A separate LT3042 (U_DAC) supplies the AD5064 VDD to avoid loading the reference output with the 4mA DAC supply current.

Vref RC Filter

R11 = 1kΩ (series from LTC6655 VOUT_F) C17 = 10µF X7R 1206 (shunt from VREF_FILT node to AGND) f_corner = 1 / (2π × 1k × 10µ) = 15.9 Hz At 160Hz: attenuation = 20dB (×10 noise reduction) At 1.6kHz: attenuation = 40dB (×100 noise reduction) // Noise at AD5064 Vref pins after filter: LTC6655 noise = 1.6µV p-p (0.1–10Hz) → passes through unattenuated Wideband noise = 7nV/√Hz → attenuated ×10 at 160Hz and beyond DAC output uncertainty = 1.6µV / 76.3µV × 1LSB ≈ 0.02 LSB
✓ C17 capacitor type matters: Use X7R 1206 (Murata GRM31 series). Avoid Y5V or Z5U — capacitance drops 60–70% at 5V DC bias, shifting the filter corner to 40–50Hz and reducing noise rejection significantly.

SPI Interface & Level Shifting

The Teensy 4.1 operates at 3.3V logic. The AD5064 requires 5V logic (VIH min ≈ 3.5V). A 74AHCT125 quad buffer translates SCLK, MOSI, and CS from 3.3V to 5V. The MISO/SDO return path uses a 10kΩ/20kΩ resistor divider (5V → 3.3V).

// 24-bit SPI word: [C3 C2 C1 C0 A3 A2 A1 A0 D15...D0] // Command = 0x3 (write and update DAC register) // Channel address: A=0x1, B=0x2, C=0x3, D=0x4 void dac_write(uint8_t ch, uint16_t code) { uint8_t buf[3]; buf[0] = (0x3 << 4) | (ch & 0xF); // cmd | address buf[1] = code >> 8; // MSB buf[2] = code & 0xFF; // LSB SPI.transfer(buf, 3); } // Usage: dac_write(0x1, laser_code); // laser Ch A: 0x13xxxx dac_write(0x2, tec_code); // TEC Ch B: 0x23xxxx

NTC Temperature Calculation (Steinhart-Hart)

float ntc_temperature(uint16_t adc_code) { float v = adc_code * 3.3f / 4095.0f; float r_ntc = 10000.0f * v / (3.3f - v); // 10k pullup float t_k = 1.0f / (log(r_ntc/10000.0f)/3950.0f + 1.0f/298.15f); return t_k - 273.15f; // °C }

Critical Layout Rules

  • LT3042 / LT3045 SET pin: surround with guard ring at OUT potential on both layers. PCB leakage of 1nA = 55µV output error on 54.9kΩ SET resistor.
  • R5 and R7 Kelvin sense resistors: the K (sense) pads must connect only to signal traces to the INA826 inputs — never shared with the power current path.
  • LTC6655-5 VOUT_S (pin 6): connect with a separate Kelvin trace directly to the AD5064 REFA/REFB pads — not back to VOUT_F through a shared trace.
  • C17 (Vref RC cap): place within 5mm of AD5064 REFA/REFB pins, GND side direct to AGND plane via.
  • TPS7A39 FBP/FBN pins: keep R1P/R2P and R1N/R2N resistors within 5mm. No switching signals within 10mm of these nodes.
  • H-bridge FETs Q3–Q6: off-board only. Flying lead J8 kept under 150mm. Heatsink thermally isolated from laser mount.
